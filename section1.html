The 4 Program Types

A program represents a single marketing initiative. It's a container with all the assets you need to make the program work.

Program Type When to Use Special Features
üìÖ Event Program Webinars, in-person events, conferences - anything with attendees at a specific date/time Webinar integration (Zoom, ON24, etc.), iPad check-in, event cap, waitlist

Email Program Features

üìö Official Documentation: Email Programs ‚Üí

A/B Testing

Email Programs support A/B testing for:

  • Subject Line (most common)
  • Whole Email (two completely different emails)
  • From Address
  • Date/Time
Winner Criteria:
‚Ä¢ Open Rate
‚Ä¢ Click Rate
‚Ä¢ Click-to-Open Rate
‚Ä¢ Engagement Score (recommended minimum 24-hour test period)
‚Ä¢ Custom Conversion (based on specific URL visits)

Recipient Time Zone

Delivers email at each recipient's local time (e.g., everyone gets it at 9 AM their time).

‚è∞ Requirement: Must approve program at least 25 hours before scheduled send.

Head Start

Processes email send 12 hours early to improve deliverability for large sends.

‚è∞ Requirement: Must approve program at least 12 hours before scheduled send.

‚ö†Ô∏è Important: Cannot use Head Start with A/B testing OR with Recipient Time Zone!

Feature Compatibility

Cannot be combined:

  • ‚ùå Head Start + A/B Testing
  • ‚ùå Head Start + Recipient Time Zone

Can be combined:

  • ‚úÖ A/B Testing + Recipient Time Zone

Engagement Program Concepts

üìö Official Documentation: Engagement Programs ‚Üí

Streams

Containers for content within an Engagement Program. A person can be in ONE stream at a time.

Content types allowed in streams:

  • ‚úÖ Emails
  • ‚úÖ Default Programs (nested programs for complex logic)
  • ‚ùå Email Programs (not allowed)
  • ‚ùå Event Programs (not allowed)
  • ‚ùå Other Engagement Programs (not allowed)

Casts

When Marketo sends content from the Engagement Program. You set the cast schedule (e.g., "Every Tuesday at 10 AM").

Cadence

Person-level setting:

  • Normal: Person receives content at next cast
  • Paused: Person does NOT receive content (even if they're a member)

Exhausted

Status when a person has received all available content in their current stream.

üéØ Tokens

üìö Official Documentation: Tokens Overview ‚Üí

1. System Tokens

Pre-defined by Marketo, always available.

Examples:

  • {{system.dateYear}} - Current year (e.g., 2025)
  • {{system.date}} - Current date
  • {{system.time}} - Current time
  • {{system.unsubscribeLink}} - Unsubscribe link
  • {{system.viewAsWebpageLink}} - "View as Web Page" link

2. Lead Tokens

Pull data from person record fields.

Syntax: {{lead.FieldName}}

Examples:

  • {{lead.First Name}}
  • {{lead.Email Address}}
  • {{lead.Company}}
  • {{lead.Lead Score}}
Default Values:
Always use default values to avoid displaying empty tokens:
{{lead.First Name:default=Friend}}
{{lead.Company:default=your company}}

3. Company Tokens

Pull data from the company/account object (when synced with CRM).

Syntax: {{company.FieldName}}

Examples:

  • {{company.Company Name}}
  • {{company.Annual Revenue}}
  • {{company.Industry}}

4. My Tokens

Custom tokens YOU create at the Program or Folder level.

Syntax: {{my.tokenName}}

üìö Official Documentation: My Tokens ‚Üí

My Token Types:

  • Text: Plain text (e.g., event name, product name)
  • Number: Numeric values
  • Date: Date values
  • Rich Text: Formatted HTML content
  • Score: Numeric score values
  • Salesforce Campaign: Link to SFDC campaign

Token Inheritance

More specific tokens override less specific ones:

Folder: {{my.region}} = "EMEA"
Program: {{my.region}} = "AMER"

Email in that program displays: AMER (program level wins)

Where Can Tokens Be Used?

  • ‚úÖ Email subject lines
  • ‚úÖ Email body content
  • ‚úÖ Landing page content
  • ‚úÖ Snippets
  • ‚úÖ Some flow step values
  • ‚ùå NOT in Smart List filters/triggers

Channels and Tags

üìö Official Documentation: Program Channels ‚Üí

Channels

Categorize program types for reporting and organization.

Examples: Webinar, Email Blast, Tradeshow, Content Syndication, Online Advertising

Each channel has defined statuses that track progression (e.g., Invited ‚Üí Registered ‚Üí Attended).

Custom Tags

Additional categorization attributes you define.

Examples:

  • Region: AMER, EMEA, APAC
  • Product Line: Product A, Product B, Product C
  • Target Audience: Enterprise, SMB, Startup

‚ö†Ô∏è Important: One program can have one value per tag type (e.g., one Region AND one Product, but not multiple Regions).

Where Channels/Tags Are Used

  • ‚úÖ Program Performance Report (can filter by channel/tags)
  • ‚úÖ Program Analyzer (if enabled)
  • ‚ùå Email Performance, Landing Page Performance, other asset-level reports

Period Costs

Track direct program costs to enable ROI calculation. Period Costs are entered in the program's Setup tab.

üìö Official Documentation: Period Costs ‚Üí

üìö Adobe Experience League: Period Costs ‚Üí

What to Include:

  • ‚úÖ Event venue rental
  • ‚úÖ Pay-per-click advertising
  • ‚úÖ Content production costs
  • ‚úÖ Vendor fees specific to the program

What NOT to Include:

  • ‚ùå Marketo subscription (overhead)
  • ‚ùå Full-time employee salaries (overhead)
  • ‚ùå Office rent (overhead)

ROI Calculation

ROI = (Revenue - Cost) √∑ Cost √ó 100%

Example:
Revenue: $50,000
Period Cost: $10,000
ROI = ($50,000 - $10,000) √∑ $10,000 √ó 100%
ROI = $40,000 √∑ $10,000 √ó 100% = 400%

Program Membership vs. Acquisition

üìö Official Documentation: Attribution (Membership & Acquisition) ‚Üí

Program Membership

When a person becomes associated with a program through:

  • Filling out a form
  • Smart campaign flow step ("Change Program Status")
  • List import
  • API

Program Acquisition

The FIRST program that brought a NEW person into your database.

Acquisition program never changes - it's always the first program.

Used for: Attribution reporting, understanding lead sources.
