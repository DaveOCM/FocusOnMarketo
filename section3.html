Asset Types Overview

üìö Official Documentation: Local vs Global Assets ‚Üí

Local vs Global Assets

Aspect Local Assets Global Assets
Location Created within a program in Marketing Activities Created in Design Studio
Availability Only available to that specific program Available to ALL programs
Use Case Program-specific assets Reusable across programs
Examples Event-specific landing page, one-time email Company logo, preference center, global footer

Use LOCAL when: Asset is specific to one program/campaign and won't be reused

Use GLOBAL when: Asset will be reused across multiple programs with centralized control

Forms

üìö Official Documentation: Creating Forms ‚Üí

Form Pre-Fill

Automatically populates form fields with known visitor data (First Name, Email, etc.)

Security Restriction: Form Pre-Fill ONLY works on Marketo landing pages with known visitors. It does NOT work on external websites for security reasons.

Progressive Profiling

üìö Official Documentation: Progressive Profiling ‚Üí

Shows different questions to returning visitors to gradually collect more information over multiple form fills.

Example: First visit asks Name/Email. Second visit asks Company/Title. Third visit asks Phone/Country.

Visibility Rules

üìö Official Documentation: Visibility Rules ‚Üí

Show/hide form fields dynamically based on answers to previous questions.

Example: If Country = "United States", show State dropdown. Otherwise, hide it.

Global Form Validation Rules

Set at Admin level to block specific email domains from ALL forms.

Example: Block free email providers like gmail.com, yahoo.com for B2B lead gen.

Form CAPTCHA

Marketo uses score-based behavioral analysis (not visual puzzles) to detect bots.

Emails

üìö Official Documentation: Creating Emails ‚Üí

Marketing vs Operational Emails

Aspect Marketing Emails Operational Emails
Respects Unsubscribed? ‚úÖ Yes - won't send ‚ùå No - will send anyway
Respects Marketing Suspended? ‚úÖ Yes - won't send ‚ùå No - will send anyway
Communication Limits? ‚úÖ Yes - respects limits ‚ùå No - bypasses limits
Use Cases Newsletters, promotions, nurture Order confirmations, password resets, shipping notifications
Important: Operational emails should ONLY be used for transactional messages that people need regardless of their subscription status.

Text Only vs HTML Emails

  • HTML Emails: Can track opens (via image pixel) and clicks
  • Text Only Emails: Cannot track opens (no image pixel), but can track clicks

Email Preheader

Summary text that appears after subject line in inbox preview (40-100 characters optimal).

Landing Pages

üìö Official Documentation: Landing Pages ‚Üí

Free-Form vs Guided Landing Pages

  • Free-Form: Drag-and-drop flexibility. You can place elements anywhere on the page.
  • Guided: Template-controlled structure. Developer creates template with defined regions; marketer fills in content.

Key Concepts

  • URL Activation: Landing page URL becomes active when the page is Approved. Before approval, URL returns 404 error.
  • Forms: Can add Marketo forms to landing pages. Forms must be Approved before adding to page.
  • Tokens: Can use tokens in landing page content for personalization (same as emails).
  • Dynamic Content: Can use Segmentations to show different content per segment (same as emails).
  • Templates: Landing pages are based on templates (either Marketo's starter templates or custom templates).

Pre-Fill on Landing Pages

Form Pre-Fill works on Marketo landing pages with known visitors. The form automatically populates with known data (Name, Email, etc.) to reduce friction.

Snippets

üìö Official Documentation: Snippets ‚Üí

Reusable content blocks that can be used in emails and landing pages.

Key Benefit: Update once ‚Üí changes everywhere the snippet is used (once approved).

Example Use Cases:

  • Email footer with social links and unsubscribe
  • Company disclaimer text
  • Event details that appear in multiple emails

Dynamic Content

üìö Official Documentation: Dynamic Content ‚Üí

What is Dynamic Content?

Content that changes based on the viewer's segment membership. Different people see different content in the same email/landing page.

Requirements

  1. Create a Segmentation (Database ‚Üí New Segmentation)
  2. Approve the Segmentation
  3. Use Segmentation in email/landing page/snippet
Remember: Segmentations are mutually exclusive - a person can be in only ONE segment. First matching segment wins (top-to-bottom evaluation).

What Can Be Dynamic?

  • ‚úÖ Text modules in emails
  • ‚úÖ Images in emails
  • ‚úÖ Snippets (entire content blocks)
  • ‚úÖ Landing page content
  • ‚ùå Forms (use Visibility Rules instead)

Default Segment

Every segmentation must have a Default segment that catches everyone who doesn't match specific segments.

Critical QA Step: Always preview by EVERY segment before sending to ensure all variations are correct!

Best Practices

  • ‚úÖ Always approve assets before using them in campaigns
  • ‚úÖ Use global assets for reusable content (footers, preference centers)
  • ‚úÖ Use local assets for program-specific content
  • ‚úÖ Test forms on Marketo landing pages (not external sites) for pre-fill
  • ‚úÖ Use Operational emails only for transactional messages
  • ‚úÖ Preview all segments when using dynamic content
  • ‚úÖ Keep preheader text between 40-100 characters
  • ‚úÖ Use snippets for content that appears in multiple places
  • ‚úÖ Use progressive profiling to reduce form length for returning visitors
