Smart Campaigns Overview

üìö Official Documentation: Smart Campaigns ‚Üí

Smart Campaigns are the engine of Marketo. They perform actions based on criteria you define.

The 3 Parts of a Smart Campaign

  1. Smart List (WHO): Defines who qualifies using filters and/or triggers
  2. Flow (WHAT): Defines what actions to perform (flow steps)
  3. Schedule (WHEN): Defines when to run (once, recurring, or activate)

Batch vs Trigger Campaigns

üìö Official Documentation: Batch vs Trigger ‚Üí

Aspect Batch Campaigns Trigger Campaigns
Smart List Contains Filters ONLY (no triggers) At least ONE trigger (can have filters too)
When They Run On schedule (once or recurring) Real-time when trigger fires
Processing Processes everyone who qualifies at runtime Processes one person at a time as they trigger
Example Use Weekly data cleanup, monthly scoring updates Send alert when form filled, score when email clicked
Key Rule: Filters only = Batch. At least one trigger = Trigger campaign.

Filters vs Triggers

Filters: Find people who match criteria (past tense)

  • Example: "Clicked Link in Email" = everyone who clicked in the past
  • Use: Batch campaigns to process groups of people

Triggers: Fire when someone does something (present tense)

  • Example: "Clicks Link in Email" = fires immediately when someone clicks
  • Use: Real-time response to actions

Common Flow Steps

üìö Official Documentation: Flow Steps ‚Üí

  • Send Email: Send a one-time email
  • Send Alert: Send internal notification to sales/operations team
  • Change Data Value: Update field values
  • Change Score: Modify lead score
  • Add to List: Add person to static list
  • Remove from List: Remove person from static list
  • Change Program Status: Update program member status
  • Wait: Pause flow execution for time period
  • Remove from Flow: Stop flow execution immediately
  • Request Campaign: Call another Smart Campaign

Wait Steps

üìö Official Documentation: Wait Steps ‚Üí

Wait Duration: Maximum 730 days (2 years)

Wait Types:

  • Duration: Wait X days/hours/minutes
  • Until Specific Date/Time: Wait until {{my.eventDate}} or specific date
  • Until Day of Week: Wait until next Tuesday at 10 AM
Critical: If you DEACTIVATE a trigger campaign with people in Wait steps, they stop processing and remain in wait. Reactivating resumes from where they left off.

Choice Steps

üìö Official Documentation: Choice Steps ‚Üí

Add conditional logic to flow steps using Choices.

Example:

Send Email:
Choice 1: If State = CA, send CA Email
Choice 2: If State = TX, send TX Email  
Default: send General Email

First matching choice executes. If no choices match, Default executes.

Request Campaign

üìö Official Documentation: Request Campaign ‚Üí

Calls another Smart Campaign from within a flow.

Requirement: The called campaign MUST have "Campaign is Requested" trigger in its Smart List.

Use Cases:

  • Reuse complex logic across multiple campaigns
  • Modular campaign design
  • Call scoring campaigns from multiple entry points

Qualification Rules

üìö Official Documentation: Qualification Rules ‚Üí

Control how often people can run through a campaign.

Rule Meaning
Once per person (default) Person can run through once, ever
Every time Person runs through every time they qualify
Once every X days Person can run through again after X days
Important: Qualification rules apply even when using Request Campaign. If called campaign is "once per person" and person already ran, they won't run again.

Remove from Flow

üìö Official Documentation: Remove from Flow ‚Üí

Immediately stops flow execution for that person. All subsequent flow steps are skipped.

Example:

Send Email A
Wait 3 days
Send Email B
Remove from Flow if Opportunity Created = True
Wait 7 days
Send Email C

If person gets opportunity after Email B ‚Üí Email C does NOT send

Best Practices

  • ‚úÖ Use Batch campaigns for scheduled processing of groups
  • ‚úÖ Use Trigger campaigns for real-time responses
  • ‚úÖ Set appropriate qualification rules (default "once per person" prevents accidental duplicates)
  • ‚úÖ Test Wait steps carefully - remember max is 730 days
  • ‚úÖ Use Request Campaign for reusable logic
  • ‚úÖ Use Remove from Flow to stop processing when conditions are met
  • ‚úÖ Use Choice steps for conditional logic within flow steps
  • ‚úÖ Don't deactivate trigger campaigns with people in Wait steps unless intentional
