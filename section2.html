The Database

üìö Official Documentation: Database Dashboard ‚Üí

The Database is home to ALL person records in your Marketo instance. Every lead, contact, customer, and prospect lives here.

Key Database Concepts

Person Record:

  • Contains all field values for an individual
  • Includes standard fields (First Name, Email, Company) and custom fields
  • Tracks complete activity history

Activity Log - Every person record has an activity log showing:

  • Email sent, opened, clicked, bounced
  • Form filled out
  • Web page visited
  • Data value changed
  • Score changed
  • Program status changed
  • Interesting moments
  • Sales activities

Static Lists

A static list is a fixed collection of people that doesn't change unless you manually modify it.

Use Cases:

  • Event attendance lists (imported from registration platform)
  • Webinar attendees (synced from webinar platform)
  • Purchased lists or list uploads
  • One-time imports for specific campaigns
  • VIP customer lists
  • Trade show booth visitors
  • Sales-provided account lists

How to Add People to Static Lists

  1. List Import: Upload CSV/Excel file and map to Marketo fields
  2. Drag and Drop: Select people and drag to list in tree
  3. Smart Campaign Flow Step: Add to List ‚Üí [Select List Name]
  4. Single Action Flow: Select people ‚Üí Person Actions ‚Üí Marketing ‚Üí Add to List

How to Remove People from Static Lists

‚ö†Ô∏è Common Mistake:
Deleting a person removes them from the ENTIRE DATABASE, not just the list!

Correct way: Use "Remove from List" flow step or single action.

Methods:

  • Smart Campaign Flow Step: Remove from List ‚Üí [Select List Name]
  • Single Action Flow: Select people ‚Üí Person Actions ‚Üí Marketing ‚Üí Remove from List

Smart Lists

A Smart List is a dynamic query that automatically updates as people meet or stop meeting the criteria.

üìö Official Documentation: Smart List Filters ‚Üí

Components:

  • Filters: Define criteria based on data values or past activities
  • Operators: is, is not, contains, starts with, greater than, etc.
  • Logic: AND/OR combinations using Advanced Filters

Common Operators

  • is: Exact match
  • is not: Does not match exactly
  • contains: Text appears anywhere in field
  • not contains: Text does not appear in field
  • starts with: Text begins with specified value
  • ends with: Text ends with specified value (e.g., email domain)
  • greater than / less than: Numeric comparisons
  • is empty / is not empty: Check if field has any value
Advanced Filters:
By default, filters use AND logic (must meet ALL criteria).
Use Advanced Filters to create OR logic: (1 OR 2) AND 3
Example: Find people in California OR with Lead Score > 50

System Smart Lists

üìö Official Documentation: System Smart Lists ‚Üí

System Smart Lists are pre-built by Marketo and appear in the Database section. They cannot be edited or deleted.

Common System Smart Lists:

  • All People: Everyone in your database
  • Unsubscribed People: People who opted out
  • Marketing Suspended: People suspended from marketing emails
  • Blocklisted: People who receive NO emails (most restrictive)
  • Email Invalid: Invalid email addresses
  • Email Suspended: Temporarily suspended (usually due to bounces)
  • Possible Duplicates: Potential duplicate records for review

Segmentations

üìö Official Documentation: Segmentations ‚Üí

Segmentations divide your database into mutually exclusive groups for personalized content.

Key Characteristics

  • Mutually Exclusive: A person can be in only ONE segment
  • Ordered Evaluation: Top-to-bottom, first match wins
  • Default Segment: Catches everyone who doesn't match other segments (required)
  • Maximum: 100 segments per segmentation
  • Used For: Dynamic Content in emails, landing pages, snippets

Creating Segmentations

  1. Database ‚Üí New Segmentation
  2. Add segments with Smart List criteria
  3. Order matters - arrange from most specific to least specific
  4. Always include a Default segment
  5. Approve the segmentation
Editing Segmentations:
To edit an approved segmentation, you must first UNAPPROVE it, make changes, then RE-APPROVE it.

Segmentation Order Example

Segmentation: "Customer Tier"
1. VIP Customers (Annual Revenue > $1M)  ‚Üê Checked first
2. Enterprise (Annual Revenue > $100K)   ‚Üê Checked second  
3. SMB (Annual Revenue > $10K)           ‚Üê Checked third
4. Default (everyone else)                ‚Üê Catches remainder

Person with $500K revenue ‚Üí Enterprise (first match)

List Import Methods

üìö Official Documentation: Import Lists ‚Üí

Import Types

Import Type What It Does When to Use
New People and Updates Creates new records AND updates existing Standard imports with mixed new/existing people
New People Only Creates only new records, skips existing Adding new leads without touching existing data
Update People Only Updates only existing records, skips new Data enrichment from vendors (updates only)
List Member Only Just adds to list, no field updates Fastest - when you only need list membership

Import Process

  1. Prepare CSV or Excel file
  2. Database ‚Üí New ‚Üí Import List
  3. Upload file
  4. Map columns to Marketo fields
  5. Select import type
  6. Select or create Static List
  7. Import
Important:
Email Address is required to match existing records. Marketo uses email as the unique identifier.

Clearing Incorrect Field Data

If you accidentally mapped data to the wrong field during import:

  1. Create Smart List with affected people
  2. Smart Campaign Flow Step: Change Data Value
  3. Set field to NULL (all caps) to clear the value

Database Best Practices

  • ‚úÖ Use Smart Lists for dynamic segmentation
  • ‚úÖ Use Static Lists for fixed point-in-time snapshots
  • ‚úÖ Always include Default segment in Segmentations
  • ‚úÖ Order Segmentation segments from most to least specific
  • ‚úÖ Use "NULL" (all caps) to clear field values
  • ‚úÖ Use "Remove from List", never "Delete" to remove from lists
  • ‚úÖ Respect Unsubscribed status (legal requirement)
  • ‚úÖ Use "List Member Only" import for speed when you only need list membership
  • ‚úÖ Validate email addresses before import to avoid Email Invalid records
